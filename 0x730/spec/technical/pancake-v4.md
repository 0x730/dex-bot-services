# Pancake V4 Technical Spec

## Current Contract

Pancake V4 means PancakeSwap Infinity. This repository does not currently own a verified Pancake Infinity route builder.

The Pancake V4 service contract relays pre-built Universal Router calldata and value generated by an upstream Pancake-compatible router. It must not synthesize Infinity swap commands from simple `tokenAddress`, `amountInBnb`, `amountInTokens`, or `fee` inputs until that builder is specified and tested.

## Acceptance Criteria

- `/pancake-v4-execute` accepts `chain`, pre-built `calldata`, and optional `valueWei` or `valueEth`.
- `/pancake-v4-buy`, `/pancake-v4-sell`, `/pancake-v4-swap`, and estimate aliases share the same pre-built calldata contract.
- Missing or invalid calldata returns HTTP 400 with a clear message before RPC configuration is required.
- The no-network contract check `npm run test:pancake-v4-calldata` passes.
- The no-network route-builder gate check `npm run test:pancake-v4-route-builder` passes.
- The localhost HTTP contract check `npm run test:pancake-v4-http-contract` passes.

## Internal Route Builder Gate

Simplified Pancake V4 route inputs must remain disabled until this repository can
produce verified Pancake Infinity Universal Router calldata.

The minimum route-builder input contract is:

- `side`: `buy`, `sell`, or `swap`.
- `currencyIn` and `currencyOut`: base route currencies; both must belong to the
  selected pool key.
- `amountIn` and `amountOutMinimum`: base-unit integer strings. Slippage belongs
  upstream until a quote source is specified.
- `recipient` and `deadline`.
- `poolKey.currency0`, `poolKey.currency1`, `poolKey.poolManager`, `poolKey.fee`,
  optional `poolKey.hooks`, and `poolKey.parameters`.
- `poolKey.parameters.tickSpacing` for CL pools or `poolKey.parameters.binStep`
  for Bin pools. Exactly one pool parameter type is allowed.

The current `@pancakeswap/v4-sdk` dependency can encode pool keys and
pool-manager swap calldata. It does not expose a verified Universal Router
planner for Pancake Infinity in this service. Therefore
`buildPancakeV4UniversalRouterCalldata` must throw a clear unsupported error and
the HTTP service must continue to require pre-built calldata.

## Rationale

Pancake Infinity pool keys include `poolManager` and encoded `parameters`; they are not Uniswap V4 pool keys. The installed `@pancakeswap/v4-sdk` exposes pool manager calldata helpers, not a Universal Router planner.
