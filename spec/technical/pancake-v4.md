# Pancake V4 Technical Spec

## Current Contract

Pancake V4 means PancakeSwap Infinity. This repository does not currently own a verified Pancake Infinity route builder.

The Pancake V4 service contract relays pre-built Universal Router calldata and value generated by an upstream Pancake-compatible router. It must not synthesize Infinity swap commands from simple `tokenAddress`, `amountInBnb`, `amountInTokens`, or `fee` inputs until that builder is specified and tested.

## Acceptance Criteria

- `/pancake-v4-execute` accepts `chain`, pre-built `calldata`, and optional `valueWei` or `valueEth`.
- `/pancake-v4-buy`, `/pancake-v4-sell`, `/pancake-v4-swap`, and estimate aliases share the same pre-built calldata contract.
- Missing or invalid calldata returns HTTP 400 with a clear message before RPC configuration is required.
- The no-network contract check `npm run test:pancake-v4-calldata` passes.
- The localhost HTTP contract check `npm run test:pancake-v4-http-contract` passes.

## Rationale

Pancake Infinity pool keys include `poolManager` and encoded `parameters`; they are not Uniswap V4 pool keys. The installed `@pancakeswap/v4-sdk` exposes pool manager calldata helpers, not a Universal Router planner.
